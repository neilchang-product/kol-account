# 環比增長率（Growth Rate）設計規格

## 概述

帳號分析頁（kol-account.html）的數據指標卡片下方顯示環比增長率，隨環比單位（日/週）切換而連動。

環比一律顯示**當期與上期的比較**，固定取資料區間末端往回推算。

---

## 核心規則

1. 環比一律是**最後的當期**與**上期**相比
2. 資料範圍必須至少符合 **2 倍的期間天數**，否則顯示 `—`
3. 不足時一律顯示 `—`，代表資料 base 不足，無環比資料

| 環比單位 | 期間天數 | 最低資料需求 | 本期 | 前期 | 公式 |
|---------|---------|-------------|------|------|------|
| 日 | 1 天 | >= 2 日 | 最後 1 天 | 倒數第 2 天 | `(本期 - 前期) / 前期 × 100%` |
| 週 | 7 天 | >= 14 日 | 最後 7 天加總 | 倒數第 8~14 天加總 | `(本期 - 前期) / 前期 × 100%` |

---

## 環比單位切換的雙重作用

切換「日 / 週」同時影響兩個區域：

1. **下方趨勢圖的粒度** — 日 = 每日一個資料點，週 = 固定 7 天切一組
2. **指標卡片的增長率** — 日 = 與昨天比，週 = 與上週比

---

## 日環比（與昨天比）

| 項目 | 定義 |
|------|------|
| 本期 | 區間最後一天（例：3/31） |
| 前期 | 區間倒數第二天（例：3/30） |
| 最低資料需求 | >= 2 日 |
| 公式 | `(本期值 - 前期值) / 前期值 × 100%` |

### 特殊情境

| 情境 | 處理 |
|------|------|
| 資料不足 2 日 | 顯示 `—` |
| 前期值 = 0 | 顯示 `—` |

---

## 週環比（與上週比）

| 項目 | 定義 |
|------|------|
| 本期 | 最後 7 天加總 |
| 前期 | 倒數第 8~14 天加總 |
| 最低資料需求 | >= 14 日 |
| 公式 | `(本期加總 - 前期加總) / 前期加總 × 100%` |

### 滾動式取法

週環比從區間末端往回滾動取 7 天，本期和前期都固定是完整 7 天：

```
假設區間 3/01 ~ 3/31：
  本期 = 3/25 ~ 3/31（7 天）
  前期 = 3/18 ~ 3/24（7 天）
```

這與趨勢圖、貼文列表的「週」切法不同（見下方說明）。

### 特殊情境

| 情境 | 處理 |
|------|------|
| 資料不足 14 日 | 顯示 `—` |
| 前期加總 = 0 | 顯示 `—` |

---

## 「週」的兩種定義（環比 vs 視覺呈現）

頁面中有兩處使用「週」的概念，定義不同但各有其合理性：

| 位置 | 週的定義 | 用途 |
|------|---------|------|
| **環比增長率**（指標卡片） | 滾動式：從末端往回取最後 7 天 vs 倒數 8~14 天 | 計算增長百分比 |
| **趨勢圖 + 貼文列表篩選** | 固定切法：從區間起始日每 7 天一組 | 視覺化分段呈現 |

### 為什麼不同

- **環比**需要的是「最近的趨勢變化」，滾動取法能精確反映最新狀態
- **趨勢圖/篩選**需要的是「穩定的分段」，固定切法讓每段邊界一致，便於對照

---

## 計算範例

### 範例 1：日環比（正常）

假設區間 3/01 ~ 3/31，指標 = Like：

```
3/30 的 Like = 8,500
3/31 的 Like = 9,200
增長率 = (9,200 - 8,500) / 8,500 × 100% = +8.2%
顯示：↑ +8.2% 與昨天比
```

### 範例 2：週環比（正常）

假設區間 3/01 ~ 3/31，指標 = Like：

```
本期: 3/25~3/31  加總 63,000
前期: 3/18~3/24  加總 56,000
增長率 = (63,000 - 56,000) / 56,000 × 100% = +12.5%
顯示：↑ +12.5% 與上週比
```

### 範例 3：資料不足（僅 4 日）

假設區間 3/28 ~ 3/31（僅 4 日）：

```
日環比：3/31 vs 3/30 → 可計算 ✓
週環比：需要 >= 14 日，僅 4 日 → 顯示 —
```

### 範例 4：資料不足（超過 1 週但不足 2 週）

假設區間 3/22 ~ 3/31（共 10 日）：

```
日環比：3/31 vs 3/30 → 可計算 ✓
週環比：需要 >= 14 日，僅 10 日 → 顯示 —

說明：雖然已超過 7 天，但週環比需要本期 7 天 + 前期 7 天 = 14 天。
      10 日只能覆蓋本期（3/25~3/31），前期（3/18~3/24）資料不完整，
      因此無法計算週環比。
```

---

## 顯示格式

```
↑ +12.5%        與上週比
↓ -3.2%         與昨天比
—                與昨天比    ← 資料 base 不足，無環比資料
```

- 正值：綠色 + `trending_up` icon
- 負值：紅色 + `trending_down` icon
- 無資料：灰色 `—`
- 「與昨天比」/「與上週比」文字靠右，顏色較淡
